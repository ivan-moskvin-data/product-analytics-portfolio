# Оптимизация агрегации логов: Извлечение лучших квалификационных баллов пользователей

### 📝 Бизнес-задача
EdTech-платформе необходимо сегментировать пользователей для автоматического открытия доступа к продвинутым когортам. Пользователи могут пересдавать квалификационные тесты. Данный микросервис анализирует сырые транзакционные логи, находит максимальный балл каждого пользователя (с учетом наиболее актуальной даты при равных баллах) и формирует очищенный JSON-файл для интеграции с CRM.

### 💡 Решение и производительность
Изначально алгоритм агрегации работал за квадратичное время O(n²), создавая узкое горлышко при росте базы пользователей. Скрипт был оптимизирован до O(n log n) за счет использования хеш-таблиц для однопроходной фильтрации логов. 

Это обеспечивает:
* Мгновенную обработку больших массивов логов тестирования без перегрузки серверов.
* Гарантированную актуальность данных в CRM.
* Защиту от падений скрипта при наличии аномалий в данных (битые строки, пустые значения).

### 🛠 Стек
* Python (Core)
* Концепции: Алгоритмическая сложность (Big O), Type Hinting, Хеш-таблицы, Обработка Edge Cases.
