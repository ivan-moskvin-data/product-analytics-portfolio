# Оптимизация аудита структуры аналитических выгрузок (Data Quality)

### 📝 Бизнес-задача
Ежедневно инженерная команда передает аналитикам гигабайты сырых данных (логи, транзакции, результаты A/B тестов) в виде вложенных ZIP-архивов. Проблема в том, что загрузка битых или аномально больших файлов может привести к падению аналитического DWH-сервера. 

Необходимо разработать утилиту для пре-аудита архива `analytics_export.zip`. Она должна визуализировать структуру директорий (сегментов и когорт) и выводить размер каждого лог-файла в читаемом формате (KB, MB, GB) с правильными отступами, чтобы аналитик мог быстро оценить полноту и корректность выгрузки.

### 💡 Решение и производительность
Изначально алгоритм расчета размера файла работал за $O(K)$ (через цикл деления), но был оптимизирован до $O(1)$ за счет использования математической функции `math.log` для мгновенного определения единицы измерения. Обход файловой структуры работает за $O(N \cdot M)$, где $N$ — количество файлов, а $M$ — глубина вложенности директорий.

Это обеспечивает:
* Мгновенную обработку и расчет размеров дампов любого масштаба без лишних ресурсоемких циклов.
* Отказоустойчивость: скрипт динамически определяет пути через `os.path`, корректно обрабатывает пустые файлы (0 B) и не падает при отсутствии или повреждении архива.

Логика протестирована локально с использованием pytest для граничных значений размеров файлов и структуры ZIP-архивов.

### 🛠 Стек
* Python (Core)
* Концепции: Big O ($O(1)$ и $O(N)$), Математическая оптимизация, Работа с файловыми системами (os, zipfile), Обработка исключений (Edge Cases).

### 🚀 Как использовать
```python
# Скрипт автоматически вычисляет свое местоположение и ищет архив рядом
from audit_script import audit_logs_archive

current_dir = os.path.dirname(os.path.abspath(__file__))
archive_path = os.path.join(current_dir, "analytics_export.zip")

# Запуск аудита логов
```
